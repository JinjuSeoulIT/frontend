# 전광판 집계 기준 (정합성 규칙)

## 목적
- 외래접수목록 화면과 전광판(`/reception/display`) 숫자가 다르게 보일 때, 원인을 빠르게 판단하기 위한 기준 문서

## 기준 요약
1. 전광판 데이터 원본: `GET /api/receptions`
2. 전광판 대상: **오늘 + 외래(OUTPATIENT)** 접수만 포함
3. 전광판 카드 헤더: `대기 n명 / 진료중 n명`으로 표시
4. 카드 본문: `WAITING`, `CALLED`, `IN_PROGRESS` 상태를 표시하고 `DONE` 계열은 제외
5. 운영 정책 확정:
`CALLED` 상태는 전광판에 **표시함**
`카드당 최대 노출 인원`은 **7명** (`DISPLAY_POLICY.maxRowsPerCard = 7`)

## 날짜 판정 규칙 (오늘 여부)
- 아래 값 중 하나라도 오늘 날짜와 일치하면 “오늘 접수”로 포함
1. `receptionNo`의 날짜 접두부 (`YYYYMMDD-...`)
2. `arrivedAt`
3. `scheduledAt`
4. `createdAt`

## 상태 판정 규칙
- 전광판 노출 상태
1. `WAITING` (또는 `대기`)
2. `CALLED` (또는 `호출`)
3. `IN_PROGRESS` (또는 `진료중`)
- 제외 상태
1. 완료/종결 계열 (`DONE`으로 정규화되는 값)

## 외래접수목록과 숫자가 달라질 수 있는 주요 케이스
1. 외래접수목록은 전체 건수를 보여주고, 전광판 헤더는 `대기/진료중` 중심으로 보여줌
2. 외래가 아닌 접수(응급/입원)가 외래접수목록 집계에 섞이는 경우
3. 날짜 판정 기준 차이로 오늘/과거 데이터 포함 범위가 달라지는 경우
4. 전광판은 카드 본문 노출을 7건으로 제한하므로 “행 개수”가 전체 건수와 다르게 보일 수 있음

## 실제 해석 예시
- 외래접수목록 `전체 26명`, 전광판 헤더 합계 `대기 22명 + 진료중 4명`
- 이 경우는 정상이며, 전광판은 상태별 분해를 보여주는 형태

## 운영 체크 순서
1. 외래접수목록 총 건수 확인
2. 전광판 헤더의 `대기/진료중` 합계 확인
3. 특정 환자 누락 의심 시:
   - `visitType`이 외래인지
   - 오늘 날짜 판정에 걸리는지
   - 상태가 완료/종결로 바뀌지 않았는지 확인
